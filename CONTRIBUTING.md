# Richtlijnen voor Bijdragen

Bedankt dat je wilt bijdragen aan dit procesvoormingsproject!

## 📋 Stappen voor Bijdragen

### 1. Fork en Clone
```bash
git clone https://github.com/rogerv85-afk/procesenovink85.git
cd procesenovink85
```

### 2. Maak een Feature Branch
```bash
git checkout -b feature/jouw-feature-naam
```

### 3. Maak je wijzigingen
- Bewerk de HTML-bestanden
- Test lokaal in je browser
- Zorg dat de structuur consistent blijft

### 4. Commit je wijzigingen
```bash
git add .
git commit -m "Add: beschrijving van wijziging"
```

### 5. Push naar je Fork
```bash
git push origin feature/jouw-feature-naam
```

### 6. Maak een Pull Request
- Beschrijf duidelijk wat je hebt gewijzigd
- Verwijzen naar gerelateerde issues (als van toepassing)

## 📐 Code Style

- Gebruik consistent HTML indentation (2 spaties)
- Voeg beschrijvende comments toe
- Zorg voor responsive design
- Test in meerdere browsers

## ✅ Checklist voor Pull Requests

- [ ] HTML valideert zonder fouten
- [ ] Wijzigingen zijn getest
- [ ] Bestand sizes zijn redelijk
- [ ] Beschrijving is duidelijk
- [ ] Commit messages volgen het format

## 🚫 Commit Message Format

```
Type: Korte beschrijving

Langere uitleg (optioneel)

Type kan zijn:
- Add: Nieuwe functionaliteit
- Fix: Bugfix
- Update: Updates en verbeteringen
- Docs: Documentatie wijzigingen
- Refactor: Code herstructurering
```
